# 🎹 全自动弹琴 JSON 制谱高级指南

欢迎使用本自动弹琴引擎！经过多次底层架构升级，目前的 V5 版本引擎不仅支持**绝对时间轴**（保证千百个音符弹下来绝不乱拍、越弹越慢），还支持了极其强大的**“千手观音”多指独立延音技术**（完美还原真实钢琴的残响与跳音）。

无论你是想要手动打谱，还是想用 AI 帮你写谱，只要看懂这篇指南，你就能让程序演奏出带有灵魂的音乐。

## 💡 一、 乐谱的基本骨架

我们的乐谱文件（`.json`）本质上是一个**动作序列**。程序会从上到下，依次执行每一个动作。
整个乐谱被包裹在一个英文中括号 `[ ]` 里，每一个动作被包裹在大括号 `{ }` 里。

**一个最标准的动作单元长这样：**
```json
{
  "notes": ["M1", "M3"], 
  "durations": [200, 400], 
  "delay": 400
}
```
它告诉程序：“**请同时按下 M1 和 M3，M1 按 200毫秒松手，M3 按 400毫秒松手。在这个动作发出的 400毫秒后，再去看下一个动作是什么。**”

---

## 📖 二、 核心参数字典（详细解析）

在 `{ }` 内部，只有三个核心参数，缺一不可：

### 1. `notes` (要按下的琴键)
这是一个**数组**。由于《无限暖暖》的琴只有 3 个八度共 21 个白键，我们制定了以下映射规则：
*   **中音区 (中间那行)：** `"M1", "M2", "M3", "M4", "M5", "M6", "M7"` （M代表Middle）
    * *兼容写法：你也可以直接写纯大写字母 `"C", "D", "E", "F", "G", "A", "B"`*
*   **高音区 (最上面那行)：** `"T1", "T2", "T3", "T4", "T5", "T6", "T7"` （T代表Top/Treble）
*   **低音区 (最下面那行)：** `"B1", "B2", "B3", "B4", "B5", "B6", "B7"` （B代表Bottom/Bass）
*   **休止符 (空拍，什么都不按)：** `"0"`

### 2. `durations` (按压时长 / 延音控制)
这是一个**数组**，里面的数字个数**必须**和 `notes` 里的音符个数一模一样！单位是毫秒(ms)。
它决定了手指在屏幕上停留的时间：
*   `80 - 150`：极短的跳音、装饰音。
*   `200 - 300`：普通的八分音符或四分音符。
*   `500 - 800`：长按拖音，制造钢琴残响。
> **注意：** 建议最大不要超过 800 毫秒，否则可能导致手机无障碍服务卡死。

### 3. `delay` (绝对延迟时间 / 节奏控制)
这是一个**整数**。它代表在这组动作下发后，距离**下一组动作**需要等待多少毫秒。
这是控制整首歌 BPM（速度）的灵魂参数。
*   假设当前歌曲速度较快，一拍为 400 毫秒：
    *   全拍：`"delay": 400`
    *   半拍（快弹）：`"delay": 200`
    *   空一拍再弹：`"delay": 800`

## 🚀 三、 基础实战案例

### 案例 1：最简单的单音旋律 (Do - Re - Mi)
要求：每个音弹 200 毫秒，中间等 400 毫秒。

```json
[
  {
    "notes": ["M1"],
    "durations": [200],
    "delay": 400
  },
  {
    "notes": ["M2"],
    "durations": [200],
    "delay": 400
  },
  {
    "notes": ["M3"],
    "durations": [200],
    "delay": 400
  }
]
```

### 案例 2：加入休止符 (停顿)
要求：弹完 Do 和 Re 之后，空一拍，再弹 Mi。

```json
[
  {
    "notes": ["M1"],
    "durations": [200],
    "delay": 400
  },
  {
    "notes": ["M2"],
    "durations": [200],
    "delay": 400
  },
  {
    "notes": ["0"],     // 0 代表不按任何按键
    "durations": [0],   // 既然不按，时长写 0 即可
    "delay": 400        // 但节奏不能断，依然要等 400 毫秒！
  },
  {
    "notes": ["M3"],
    "durations": [200],
    "delay": 400
  }
]
```

## 🔥 四、 高阶魔法案例

### 案例 3：普通和弦 (多根手指同时按)
要求：同时按下低音 Do (`B1`) 和 中音 Do (`M1`)，一起按 300 毫秒后同时松手。

```json
[
  {
    "notes": ["B1", "M1"],      // 数组里写两个音符
    "durations": [300, 300],    // 两个手指都按 300 毫秒
    "delay": 600
  }
]
```

### 案例 4：千手观音 / 错位断奏 (极度逼真！)
**这是本引擎最强大的功能。** 
要求：左手低音保持长按制造残响（600毫秒），右手高音只是轻轻点一下（100毫秒）。这能制造出极强的层次感。

```json
[
  {
    "notes": ["B1", "T3"],      // 同时按下低音Do 和 高音Mi
    "durations": [600, 100],    // 低音按住 600ms，但高音只按 100ms 就立刻松开！
    "delay": 400                // 400ms 后立刻开始下一个动作
  },
  {
    "notes": ["T4"],            // 低音B1还在按着的时候，右手立刻去弹高音Fa
    "durations": [100],
    "delay": 400
  }
]
```

## ⚠️ 六、 常见报错排查 (排雷指南)
如果你写出来的谱子加载失败，99% 是因为犯了以下几个 JSON 语法错误：
1.  **标点符号不是英文状态：** 所有的双引号 `""` 和逗号 `,` 必须是英文半角！千万别打成中文的全角 `”，`。
2.  **最后一个元素多了逗号：** 在一个中括号 `[]` 或大括号 `{}` 的最后一个元素后面，**千万不能加逗号**。
3.  **数量不匹配：** `notes` 里写了 3 个音符，但 `durations` 里只写了 2 个时间。程序会因为找不到第三根手指的时间而崩溃。
